# PRD – 솥밥 매장 예약 관리 MVP

## 1. 프로젝트 개요

### 1.1 배경

현재 솥밥 매장의 예약 관리는 종이 장부를 통해 수기로 이루어지고 있다. 이 방식은 예약 현황 공유가 어렵고, 중복 예약이나 누락이 발생할 가능성이 높다.

### 1.2 목적

* 수기 예약 관리 완전 대체
* 예약 현황을 실시간으로 공유 가능한 웹 기반 시스템 구축
* 1시간 내 개발 가능한 최소 기능 제품(MVP) 구현

### 1.3 개발 환경

* 개발 도구: Windsurf (바이브 코딩)
* 플랫폼: 웹 (모바일 우선)
* 사용 대상: 매장 직원 (카운터 / 주방)

---

## 2. MVP 범위 정의

### 2.1 반드시 포함할 기능 (In-Scope)

* 예약 등록
* 예약 리스트 조회
* 날짜별 예약 필터링 (오늘 / 내일 / 전체)
* 예약 상태 변경
* 실시간 예약 현황 공유

### 2.2 제외 기능 (Out-of-Scope)

* 고객 로그인 및 회원 관리
* 온라인 결제
* 고객용 예약 페이지
* 자동 문자/카카오톡 알림
* 좌석 배치 시각화
* 복잡한 권한 관리

---

## 3. 핵심 사용자 시나리오

### 시나리오 1: 카운터 직원 예약 등록

1. /admin 페이지 접속
2. 예약 정보 입력
3. 저장 버튼 클릭
4. 즉시 예약 리스트에 반영

### 시나리오 2: 주방 예약 현황 확인

1. /view 페이지 접속
2. 오늘 예약 리스트 확인
3. 상태 변경 여부 실시간 확인

---

## 4. 기능 상세 명세

### 4.1 예약 등록 기능 (/admin)

**입력 필드**

* 날짜 (date)
* 시간 (time)
* 인원 수 (number)
* 예약자 이름 (string)
* 전화번호 (string)
* 메모 (string, optional)

**동작**

* 저장 시 DB에 예약 정보 생성
* 저장 즉시 리스트에 반영

---

### 4.2 예약 리스트 조회

**공통 기능**

* 시간 순 정렬
* 한 줄 = 하나의 예약

**필터**

* 오늘 예약
* 내일 예약
* 전체 예약

**상태 값**

* reserved (예약)
* done (방문 완료)
* cancelled (취소 / 노쇼)

---

### 4.3 예약 상태 관리

* 상태 변경 버튼 제공
* 상태별 시각적 구분

  * 예약: 기본 색상
  * 완료: 회색
  * 취소: 연한 빨강

---

### 4.4 실시간 공유

* URL 기반 접근
* /view 페이지는 읽기 전용
* 로그인 없이 접근 가능 (또는 단순 PIN 적용 가능)
* 새로고침 시 최신 데이터 반영

---

## 5. 화면 구성

### 5.1 페이지 구조

```
/admin
- 예약 추가 폼
- 예약 리스트 (수정 가능)

/view
- 예약 리스트 (읽기 전용)
```

### 5.2 사용 예시

* 카운터: /admin (예약 입력 및 관리)
* 주방 태블릿: /view (예약 현황 확인)

## 7. 기술 선택 가이드 (권장)

* Database: Supabase (속도 및 설정 최소화)
* Auth: 사용하지 않음 (MVP 기준)
* UI: 모바일 우선, 큰 버튼과 단순한 레이아웃

---

## 8. 개발 타임라인 (1시간 MVP)

| 시간     | 작업 내용              |
| ------ | ------------------ |
| 0–10분  | 프로젝트 생성, DB 테이블 생성 |
| 10–25분 | 예약 추가 폼 구현         |
| 25–40분 | 예약 리스트 및 필터 구현     |
| 40–50분 | 상태 변경 및 삭제 기능      |
| 50–60분 | 모바일 최적화 및 실사용 테스트  |

---

## 9. UX 원칙 (매장 기준)

* 한 손 조작 가능
* 오늘 예약이 항상 상단
* 텍스트 가독성 최우선
* 불필요한 애니메이션 및 장식 제거

---

## 10. 향후 확장 아이디어 (Post-MVP)

* 하루 최대 수용 인원 초과 경고
* 특정 시간대 자동 강조 표시
* 예약 데이터 엑셀 다운로드
* 카카오톡 알림 연동
* 고객용 예약 페이지 분리

---

## 11. 성공 기준

* 종이 예약 장부 완전 제거
* 직원 누구나 5초 내 오늘 예약 파악 가능
* 모바일 환경에서 무리 없이 사용 가능

---

**핵심 원칙:**

> 종이 장부를 그대로 웹으로 옮긴다. 그 이상도, 그 이하도 아니다.
